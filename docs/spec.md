# T字路での車両旋回シミュレーション 仕様書

## 概要
垂直なT字路（縦道と水平道）で、箱型車両が前輪操舵（フロントホイールで旋回）により曲がれるかを判定・可視化するWebシミュレーション。

目的:
- 道幅（縦・横）や車両パラメータを変化させて、衝突せずに曲がれる開始位置の領域を求める
- その領域内から任意の開始位置を選び、車両の通過軌跡をアニメーション表示
- 内輪差・外輪差を計算し、運転技能マージンを反映した衝突判定を行う

## 用語・単位
- 単位: メートル（m）、角度: 度（°）
- 座標系: 2D平面 (X, Y)。初期ではキャンバス中心を原点にせず、画面左上を (0,0) とするが実装で調整可。

## パラメータ
### 道パラメータ
- 接続角度: デフォルト 90°（縦道と水平道の角度）
- 道幅A: 縦道路の幅（m）
- 道幅B: 水平道路の幅（m）

道路形状: 縦道はT字の“幹”となり、水平道が上側または下側に接続する形（本仕様では水平道が上側に接続する垂直T字路を想定）。実装では接続角度はパラメータ化可能。

### 車両パラメータ（箱型）
- 車幅 W （m）
- 車長 L （m）
- 前輪位置 from front: 前方端から前輪（前アクスル）の位置までの距離 df (m)。つまり、車両前端 → 前輪中心までの縦方向オフセット。
- ホイールベース WB (m): 前軸と後軸の距離
- 最小回転半径 Rmin (m): 前輪最大舵角時の最小旋回半径（前輪中心を基準とした曲線半径）
- 走行方向: 前進のみ（後退は今回扱わない）
- 車両は矩形（中心・向きから4頂点を求める）として衝突判定を行う

※ トレッド（左右の車輪距離）は車幅に含めて近似する（トレッド = W - マージン）。内外輪差計算にはトレッド/2 を使用。

### 運転技能マージン
- スキルマージン m_skill (m): 道幅の余裕をさらに狭めるため、車体外郭にこの値を足して当たり判定を厳しくする（または車両サイズに＋して扱う）

### 進入方向パラメータ（4パターン）
進行方向ごとに判定を行う。各パターンは "進行方向→曲がる方向" の組み合わせ:
1. 上から左 (車が画面上側から進入し左へ曲がる)
2. 上から右
3. 左から上
4. 右から上

（他の方向も拡張可能）

## 物理 / 運動学モデル
フロント操舵の単純なキネマティック・バイシクルモデルを採用。
- 状態: (x, y, θ) — 前輪中心または車体中心のどちらを基準にするかは実装で一貫させる（以下は前輪中心を基準にした表記を含む）
- 入力: 前輪舵角 δ (rad)、前進速度 v (m/s)

連続-time kinematics:
- dx/dt = v * cos(θ)
- dy/dt = v * sin(θ)
- dθ/dt = v / WB * tan(δ)

回転半径 R と舵角の関係:
- R = WB / tan(δ)
- 与えられた Rmin に対して最大舵角 δ_max = atan(WB / Rmin)

車体矩形の計算:
- 基準点を前輪中心から車体幾何でオフセットして車体中心や車体前端を求め、回転行列で4頂点を計算する。

内輪差/外輪差:
- 前輪の曲率中心を Rc = R
- 左右車輪の半径: R_left = Rc ± track/2（進行方向や曲がりの向きによる符号注意）
- 内輪の半径 = R - track/2、外輪 = R + track/2

※ 算出される軌跡は前輪中心軌跡・後輪中心軌跡・車両外郭（矩形）の各頂点軌跡を取る。

## 衝突判定
- 道路領域: 各道幅から道路ポリゴン（通行可能領域）を作る。T字路は縦路ポリゴン ∪ 水平路ポリゴンで表す。
- 車両衝突: 各タイムステップで車両矩形（頂点）と道路ポリゴンの包含/交差をチェック。
- マージン適用: `m_skill` を車両外郭に加算して衝突判定を厳しくする（または道路境界を内側にm_skillだけ縮小して扱う）。
- 衝突＝外郭の任意頂点が道路ポリゴン外に出る、または頂点または辺が境界と交差する。

判定の閾値: 衝突判定は「どの時刻でも道路外に出ない」ことを満たせば成功とする。

## 開始位置領域の探索
- 目的: 指定された進入方向に対して、車両の開始位置 (x0, y0) のうち、事前に定義した開始向き（車両が進行する向き）で、最大舵角操作（あるいは滑らかな舵角プロファイル）を使って曲がりきれるかを判定し、成功する開始位置の集合（領域）を求める。

- 探索方法（初版）:
  1. 進入方向ごとに開始領域（矩形）を定義（例: 道路端から一定距離の範囲）
  2. 格子サンプリング（解像度はUIで調整）で各サンプル位置を取り、一定の初期姿勢（向き）でシミュレーションを行う
  3. 操作ポリシー: 最短半径で回るため`δ = δ_max`を固定して前進する単純戦略でまず判定する。必要なら`δ(t)`を最適化する（後段）
  4. 衝突がなければそのサンプルは成功とする

- 出力:
  - 成功サンプルの割合・面積（サンプル分解能に依存）
  - 成功領域を塗りつぶして可視化

## シミュレーション出力（UI）
- 成功領域の表示（色分け: 成功 = 緑、失敗 = 赤）
- 選択した開始位置からの軌跡アニメーション（前輪・後輪・車体外郭を表示）、軌跡に内輪差・外輪差曲線をオーバーレイ
- 成功領域の面積（m^2換算、またはピクセルベースの面積）と、領域のXY範囲（バウンディングボックス）
- 各種パラメータをスライダーで変更可能: 道幅A/B、車幅・車長・WB・df・Rmin・m_skill、サンプリング解像度
- 進入方向の切替（4パターン）

## 実装方針（第一段階）
- 技術スタック: HTML + CSS + JavaScript（ES6） + Canvas または SVG（推奨 Canvas for swept-area animation）
- ファイル構成（簡易）:
  - index.html
  - style.css
  - app.js
  - libs/geometry.js（ベクトル/回転/ポリゴン交差）

### 主要機能の実装順
1. キャンバス上にT字路を描画、パラメータ操作UI作成
2. 車両矩形を与え、与えた姿勢で車体を描画できるようにする
3. バイシクルモデルを実装し、一定舵角での軌跡を描く
4. 衝突判定ルーチンを実装（各タイムステップでポリゴン交差）
5. サンプリング探索で成功領域を計算・表示
6. 開始位置から軌跡アニメーション（選択して再生）
